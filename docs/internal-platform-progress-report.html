Internal Report

Emorya Gamify Internal Platform Progress Report

This is an internal product/status summary of what Emorya Gamify is becoming, how it works, where it is now, and what still needs to be finished.

This is not a pitch deck. It is a practical internal reference for product, operations, partnerships, and the wider team.

What The Platform Is

Emorya Gamify is a quest, progression, reward, and campaign platform that sits on top of real user actions.

In the Emorya version, it is designed to:

  • onboard users into the Emorya ecosystem
  • guide them from simple awareness actions into real product use
  • reward repeat behavior and progression
  • create referral, retention, and UGC loops
  • connect actions to wallet-linked reward readiness

The platform is also being structured so the same underlying system can be reused for:

  • collaborative campaigns with partner projects
  • branded campaign surfaces for MultiversX ecosystem projects
  • a possible SaaS-style white-label product for other ecosystem teams

What It Does In Practice

At a practical level, the platform combines:

  • onboarding quests
  • daily / weekly habit quests
  • referral and community growth quests
  • premium conversion quests
  • creator / UGC quests
  • admin moderation and review tooling
  • reward and settlement preparation
  • leaderboard and progression surfaces

The product is not just a one-off campaign page. The intention is a repeat-use engagement layer that can power:

  • Emorya user activation
  • repeat campaign loops
  • partner onboarding
  • community-driven growth
  • wallet and reward readiness

Multi-Brand / Partner Use

One of the most important capabilities already built is the brand skin system.

That means:

  • the underlying product stays the same
  • the live public skin can be switched by a super admin
  • normal users only see the published skin
  • the platform can present as Emorya, MultiversX, xPortal, and future brands

This turns the product from one branded Emorya tool into one reusable campaign and progression system that can be presented as native to different ecosystem projects.

How It Can Be Used

1. Emorya’s own growth and retention platform

This is the main current use case.

  • create account
  • connect wallet
  • convert first calories
  • repeat calorie / streak challenges
  • premium upgrade prompts
  • referral growth
  • review / trust-building actions
  • creator and UGC challenges

2. Collaborative campaign layer

The same stack can run joint campaigns with partner projects.

  • wallet onboarding
  • ecosystem quests
  • campaign packs
  • external platform verification
  • referral and distribution mechanics

3. SaaS / white-label ecosystem tool

If developed further, the platform can be positioned as a reusable engagement and campaign system for other MultiversX ecosystem teams.

  • brand skin swap
  • project-specific quest sets
  • custom campaign packs
  • external verification
  • moderation and reward operations

  • PostgreSQL-backed product state is in place
  • real auth, sessions, and wallet linking are in place
  • admin review and moderation tooling are working
  • branded theme / skin architecture is working
  • public skin publishing by super admin is working
  • Emorya quest framework is much stronger than before
  • custom quest authoring is now much more powerful
  • multiple quest verification types are supported
  • campaign pack and admin operations surfaces already exist

Strategic Direction From Here

We are not treating this as a major platform pivot.

The current direction is:

  • most of the platform we need is already there
  • the next meaningful layer is deeper integration with real Emorya app actions
  • we want more real in-Emorya interactions, completions, and milestone moments to flow into Gamify
  • the aim is stronger psychology, stronger habit formation, and stronger product connection

So the next major theme is deeper real app-to-gamification connection, not a broad rebuild or a brand-new product concept.

What Is Still To Finish

1. Deeper in-Emorya product event integration

  • more real app actions becoming quest triggers
  • more completion states tied to actual Emorya behavior
  • more milestone moments tied to real product use
  • more psychologically meaningful reward feedback

2. Final launch hardening and hosting readiness

  • final hosted env setup
  • final smoke tests
  • production URL and ownership settings
  • wallet-connect production confirmation

3. Broader partner quest mapping

The portability system now exists, but partner quest sets still need continued refinement as real collaborations get defined.

4. Milestone-share feature activation

The placeholder quests are seeded, but the actual share-on-celebration product flow still needs to be built.

Current Internal Summary

The product is now best understood as:

  • a reusable engagement and progression platform
  • already capable of powering Emorya
  • already capable of branded partner presentation
  • already capable of custom, verified, multi-step campaign quests
  • not finished, but clearly beyond concept stage

The biggest remaining value is no longer “can we build this platform?”

It is how deeply we connect it to real in-Emorya product behavior, how effectively we use it for campaigns and partner rollouts, and how cleanly we prepare it for live use.
