# UI And Brand Pass Plan

## Goal

Bring the product from a strong systems prototype to a polished, coherent Emorya gamification experience without blurring its role as a supporting acquisition, entertainment, and retention layer.

## Positioning Guardrail

- Emorya Gamification should feel high-value and intentional.
- It should not visually or narratively overpower the core Emorya health product.
- The design should communicate:
  - fun
  - momentum
  - reward clarity
  - trust
  - ecosystem alignment

## Phase 1: Brand Foundations

1. Add real local brand font assets.
2. Replace temporary typography fallbacks with a defined type scale.
3. Standardize spacing, radii, shadows, and badge treatments across:
   - auth
   - dashboard
   - leaderboard
   - profile
   - admin

## Phase 2: Reward Clarity

1. Tighten the visual hierarchy around:
   - XP
   - premium lift
   - token payout states
   - reward programs
   - source lane context
2. Make payout state transitions feel easier to scan:
   - queued
   - approved
   - processing
   - settled
3. Improve receipt and settlement timeline readability for both users and admins.

## Phase 3: Funnel Identity

1. Give each live lane a more deliberate visual framing:
   - direct
   - zealy bridge
   - optional galxe live lane
   - optional taskon live lane
2. Keep bridge-mode copy and visuals distinct from true split-lane mode.
3. Make premium CTAs feel contextual rather than repeated boilerplate.

## Phase 4: Admin Simplification

1. Reduce visual density in admin without losing operational depth.
2. Group related controls more clearly:
   - payout controls
   - reward programs
   - campaign operations
   - templates
   - review queue
3. Make the most important operational warnings more prominent than secondary metrics.

## Phase 5: Economy Presentation

1. Create a more intentional visual language for:
   - XP-first progression
   - token overlay rewards
   - partner asset programs
   - campaign pack health
2. Make it obvious that:
   - XP is the core progression currency
   - tokens are configurable reward rails layered on top

## Recommended Implementation Order

1. Fonts and typography tokens
2. Shared card/button/badge cleanup
3. Dashboard and leaderboard reward hierarchy
4. Profile payout and receipt polish
5. Admin surface simplification
6. Final lane-specific campaign visuals

## Definition Of Done

- The product feels visually consistent across all main surfaces.
- Reward information is easier to understand at a glance.
- Admin operations remain dense but not chaotic.
- Bridge-vs-live-lane behavior is visually legible.
- The interface feels like Emorya, but still clearly as a gamification layer within the broader ecosystem.
