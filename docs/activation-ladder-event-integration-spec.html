Internal Spec

Activation Ladder Event Integration Spec

Purpose: let Emorya app or backend send trustworthy product events into Gamify so onboarding and activation quests can complete automatically from real user behavior.

What this enables A smoother activation ladder where account creation, profile completion, wallet connection, and first real actions complete automatically instead of relying on manual proof.
What this avoids Friction, fake proof, manual admin review for core onboarding, and weak “I did it, trust me” quest flows.

Core Objective

  • Use real Emorya product behavior to complete activation quests.
  • Make onboarding feel like one intentional activation ladder.
  • Derive full activation completion inside Gamify once prerequisite steps are complete.

Recommended Event Set

Event Completes Notes
app_opened_first_time open-the-app-for-the-first-time First recorded real app session.
account_created create-emorya-account One of the most important activation steps.
profile_completed complete-your-profile Requires agreed profile completeness rule.
starter_setup_completed confirm-your-starter-setup Could map to onboarding checklist completion.
wallet_connected connect-your-xportal-wallet Critical for reward readiness.
reward_path_viewed view-your-emrs-reward-path First recorded visit only.
first_calorie_conversion_completed convert-your-first-calories Main activation proof point.

Suggested Event Payload

{
  "event": "profile_completed",
  "userId": "uuid",
  "occurredAt": "2026-04-02T12:00:00.000Z",
  "source": "emorya-app",
  "metadata": {}
}

Minimum Requirements

  • A stable shared user identifier between Emorya and Gamify.
  • Reliable event emission when the underlying product action really happens.
  • Idempotent handling so repeated events do not double-complete or double-award.
  • An authenticated server-to-server or trusted client-to-server delivery path.

Gamify Integration Shape

  • Gamify already exposes POST /api/app-events.
  • Emorya app or backend sends supported activation events to that endpoint.
  • Gamify maps those events to activation quests and auto-completes them.
  • complete-the-full-activation-ladder should remain derived inside Gamify once all prerequisite quests are complete.

Questions For The App Team

  • Can the app or backend emit reliable events for account creation, profile completion, wallet connection, reward-path view, and first calorie conversion?
  • What exact user identifier can we share with Gamify?
  • Does the backend already store these states or trigger handlers when they happen?
  • Can the backend call an authenticated Gamify endpoint directly?
  • Which of these events are definitely available now versus later?
Minimum viable version
Phase 1 can be very small: account_created, profile_completed, wallet_connected, reward_path_viewed, and first_calorie_conversion_completed. That would already make the activation ladder feel much more real.

Short conclusion: if the Emorya app or backend can emit a small set of trusted product events tied to a stable user identity, the activation ladder can become mostly automatic using infrastructure that already exists in Gamify.
