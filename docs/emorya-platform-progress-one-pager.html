What Is Already In Place

Built-in login and identity

The platform already supports native account creation and sign-in with a real identity model.

  • Email/password authentication is live
  • Signed HTTP-only cookie sessions are in place
  • MultiversX wallet linking is built into the same user identity layer
  • Profile, leaderboard, achievements, and dashboard all run on the same session model

Activation and quest system

The Emorya journey is now structured around activation, progression, and repeat use.

  • Activation-first quest board is in place
  • Custom tasks and guided quest creation are already built for admins
  • Reusable quest templates and task blocks are supported
  • Proof upload, text submission, wallet checks, API checks, and manual review are all supported

Reward types and token registry

The reward layer is already structured for real token-based campaign operations.

  • Reward asset registry for accepted payout tokens is built
  • Reward programs can map assets to live payout logic
  • Direct rewards, redemption, referral rewards, premium rewards, and ambassador rewards are supported
  • Reward eligibility logic and settlement operations already exist in the admin layer

Skins and reskinnable functionality

The platform is already operating as a reusable shell rather than a one-brand prototype.

  • Emorya, xPortal, and MultiversX skins are ready
  • The same product can be re-presented with different brand themes
  • Published brand theme control already exists
  • This makes the platform white-label ready for ecosystem use and partner campaigns

Operations and Control Layer

Segmented and secure admin access

The admin surface has been tightened significantly for launch posture.

  • No admin links are exposed in the public-facing user shell
  • The public /admin route is no longer the live entry point
  • Admin access has moved behind a non-public control-room route
  • Database-backed admin and super-admin role checks remain active

Operational tooling already built

The platform is not just front-end complete; the operating layer is also in place.

  • Review queue and moderation flow are working
  • Quest authoring and template management are built
  • Reward assets and reward programs are manageable in admin
  • Payout operations and settlement surfaces already exist for controlled rollout

What This Means In Practice

For partner use
